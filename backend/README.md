# UniSync - Smart Campus Operations Hub

This project is a Monolithic Application built for the IT3030 Programming Applications and Frameworks assignment.
It implements **Module E - Authentication & Authorization**.

## Tech Stack
* **Backend:** Spring Boot 3, Spring Security, OAuth 2.0 (Google), JWT, JPA, MySQL
* **Frontend:** React (Vite JS), Axios, React Router Dom

## Prerequisites
* Java JDK 21
* Node.js v20+
* MySQL Server (running on `localhost:3306`)
* Maven

## Setup & Configuration

### 1. Database Setup
Ensure you have MySQL running and create a database named `unisync`:
```sql
CREATE DATABASE unisync;
```
*(The tables `users` will be auto-generated by Hibernate `ddl-auto=update` when the application starts)*

### 2. Google OAuth2 Configuration
You must provide a Google OAuth2 Client ID and Client Secret in `backend/src/main/resources/application.yml`. Or via environmental variables.
Go to [Google Cloud Console](https://console.cloud.google.com/), create credentials for an OAuth Client ID (Web Application), and set the redirect URI to:
`http://localhost:8080/login/oauth2/code/google`

Open `backend/src/main/resources/application.yml` and replace:
```yaml
client-id: YOUR_GOOGLE_CLIENT_ID
client-secret: YOUR_GOOGLE_CLIENT_SECRET
```

### 3. Running Backend (Spring Boot)
From the root directory:
```bash
cd backend
mvnw clean spring-boot:run
```
*(The backend runs on `http://localhost:8080`)*

### 4. Running Frontend (React)
From the `frontend` directory:
```bash
cd frontend
npm install
npm run dev
```
*(The frontend runs on `http://localhost:5173`)*

## Features Implemented
* Google OAuth 2.0 Sign in.
* User data persistence to MySQL.
* JWT Generation upon successful Google Auth.
* Axios interceptor for appending JWT `Bearer` token.
* `AuthContext` to manage global session & roles securely.
* PrivateRoutes to manage routing (e.g. `USER` and `ADMIN` routing rules).
